## 1. 基础设计原则

### 1.1 色彩与反馈层级

| 层级 | 颜色 | 用途 |
|------|------|------|
| 主色 | #7c5cfc（紫色） | 主要操作、选中态、链接 |
| 成功 | #28a745（绿色） | 归档完成、操作成功 |
| 警告 | #f0ad4e（黄色） | 确认提示、草稿待处理 |
| 错误 | #dc3545（红色） | 操作失败、异常提示 |
| 文字主要 | #1a1a2e | 标题、正文 |
| 文字次要 | #666 | 辅助信息 |
| 文字弱化 | #999 | 占位符、disabled |
| 背景 | #f5f5f8 | 页面主背景 |
| 卡片 | #ffffff | 内容卡片 |
| 分割线 | #eaeaef | 分隔 |

### 1.2 交互一致性规则

- **左键点击**：触发操作
- **右键点击**：显示上下文菜单（作品库卡片、版本管理分支）
- **双击**：无特殊行为，避免与单击混淆
- **拖拽**：三栏分隔条拖拽调整宽度
- **快捷键**：Cmd/Ctrl+S 保存 / Enter 发送消息 / Esc 关闭弹窗

## 3. 首次启动与引导

### 3.1 首次启动判断

应用启动时检测 `config.json` 是否存在且包含有效 API Key：

```
[启动]
  ├─ config.json 不存在  → 弹出设置弹窗强制引导
  ├─ API Key 为空        → 弹出设置弹窗引导
  └─ API Key 有效        → 进入作品库（默认页）
```

### 3.2 引导流程交互

```
┌─────────────────────────────────────┐
│  ⚙️ 设置                             │
│                                      │
│  ┌─────────────────────────────────┐│
│  │ LLM Provider                    ││
│  │ ┌─────────────────────────┐     ││
│  │ │ Claude (Anthropic)    ▼ │     ││
│  │ └─────────────────────────┘     ││
│  │                                  ││
│  │ API Key                          ││
│  │ ┌─────────────────────────────┐  ││
│  │ │ ●●●●●●●●●●●●●●●●●●●●●●●●  │  ││
│  │ └─────────────────────────────┘  ││
│  │                                  ││
│  │ [测试连接] ── 成功 ✅ / 失败 ❌   ││
│  │                                  ││
│  │ 主题：○ 亮色  ○ 暗色  ○ 跟随系统 ││
│  │                                  ││
│  │         [取消]     [保存并开始]   ││
│  └─────────────────────────────────┘│
└─────────────────────────────────────┘

首次启动时：
1. 未配置 → 打开设置弹窗，背景半透明遮罩
2. 用户填写 API Key
3. 点击「测试连接」→ 显示结果
4. 点击「保存并开始」→ 关闭弹窗 → 进入作品库
5. 点击「取消」→ 弹窗关闭，但页面上方显示黄色横幅
   "⚠️ 请配置 API Key 以使用 AI 功能"
```

### 3.3 横幅引导（未配置 Key 时）

```
┌─────────────────────────────────────────────┐
│ ⚠️ 请先在设置中配置 API Key，才能使用 AI 功能  │  ← 黄色横幅
│                                      [去设置]│
├─────────────────────────────────────────────┤
```

- 横幅可关闭（点击 ✕）
- 点击「去设置」跳转到设置弹窗
- 用户未配置 Key 时使用任何 AI 功能（续写、审核、世界观生成等），都会：

```
[用户点击续写] → 弹出 Toast "请先配置 API Key"
                 ↓ 2 秒后自动跳转设置弹窗
```

│                                                      │

### 4.2 页面区域划分

| 区域 | 内容 | 交互 |
|------|------|------|
| 顶部操作栏 | 标题 + 「导入」「新建」按钮 | 按钮触发弹窗 |
| 卡片网格区 | 书籍卡片列表 | 点击进入创作台；右键菜单操作 |
| 空状态区域 | 空引导（无书时显示） | 按钮触发展开 |

### 4.3 卡片交互

**默认态**：
- 显示封面 emoji + 书名 + 类型标签 + 章节数 + 字数 + 进度条
- 悬停：边框变紫 + 上移 2px + 阴影

**右键菜单**：
```
┌──────────────────┐
│  重命名           │
│  复制书籍         │
│  ────────────     │
│  删除 🗑          │  ← 红色文字
└──────────────────┘
```

**删除确认**：
```
┌─────────────────────────────────┐
│  确定删除「星辰之痕」吗？           │
│  删除后所有内容（章节、设定、Git     │
│  历史）不可恢复。                   │
│                                  │
│          [取消]    [确认删除]      │  ← [确认删除] 为红色
└─────────────────────────────────┘
```

### 4.4 组件状态

**卡片**：

| 状态 | 表现 |
|------|------|
| 默认 | 白底+1px 灰边框 |
| 悬停 | 紫边框+阴影+上移 |
| 点击 | 0.1s 缩小至 0.98 倍再恢复 |
| 导入中 | 半透明遮罩 + 进度条动画 |

**「新建」按钮**：

| 状态 | 表现 |
|------|------|
| 默认 | 紫色背景 + 白色文字 |
| 悬停 | 深紫背景 (#6a48e8) |
| 点击 | 0.1s 缩小 |
| 弹窗已打开 | 置灰 disabled |

### 4.5 异常流

| 场景 | 行为 |
|------|------|
| 加载书库失败 | 显示 Toast"加载失败" + 重试按钮 |
| 导入目录无 .md 文件 | Toast"所选目录中没有找到 Markdown 文件" |
| 删除时文件系统异常 | Toast"删除失败，请检查文件权限" |
| 导入时文件冲突 | Toast"部分章节已存在，已跳过重复文件" |

### 4.6 动效

| 场景 | 动效 |
|------|------|
| 页面进入 | 卡片从底部淡入升起（stagger：每张间隔 60ms）|
| 新建成功 | Toast 从顶部滑入，2 秒后滑出 |
| 跳转到创作台 | 页面从右向左滑入 |

### 5.2 页面区域划分（三栏可拖拽）

| 区域 | 宽度 | 内容 |
|------|------|------|
| 文件树（左侧） | 默认 240px，可调 160-400px | 章节 / 草稿 / 知识文件三层结构 |

### 5.3 文件树交互

**文件层级结构**：

```

    ├── 🎨 style_fingerprint.md

    ├── 📋 chapter_outline.md

    ├── 💡 brainstorm.md

**交互规则**：

| 操作 | 行为 |
|------|------|
| 左键单击文件 | 在编辑器加载该文件内容 |
| 当前选中文件 | 紫色背景高亮 |
| 草稿文件（draft） | 右侧显示黄色「📝 待处理」标签 |

### 5.4 编辑器交互

**工具栏**：

```
[B] [I] [U]  [H1] [H2]  [🔗] [📷]  [◀] [▶]    编辑 | 预览
 ↑粗体 ↑斜体 ↑下划线 ↑标题 ↑链接 ↑图片 ↑撤销 ↑重做    ↑切换
```

**编辑/预览切换**：

| 模式 | 说明 |
|------|------|
| 编辑 | WYSIWYG 所见即所得，工具栏可用 |
| 预览 | 只读渲染模式，Markdown 渲染为格式化文本，工具栏隐藏 |

**保存策略**：

| 触发方式 | 行为 | 反馈 |
|----------|------|------|
| Ctrl+S / Cmd+S | 立即写入文件系统 | 顶部栏显示「已保存」文案 2 秒后消失 |
| 自动保存 | 内容变更后 30s 无操作 → 自动写入 | 静默保存，无反馈 |
| 切换文件时 | 自动保存当前文件再加载目标文件 | 静默保存 |

#### 编辑器状态

| 状态 | 表现 |
|------|------|
| 加载中 | 骨架屏（灰色占位块） |
| 正常编辑 | 白底 + 文字可编辑 + 工具栏可用 |
| 只读模式 | 编辑器灰底，工具栏禁用 |
| 保存中 | 顶部栏显示「保存中…」 |
| 保存失败 | Toast「保存失败，请检查磁盘空间」 |

### 5.5 AI 聊天面板交互

**消息列表**：

```
┌──────────────────────────────┐
│  消息历史 (可滚动)            │
│                              │
│  ┌────────────────────────┐  │
│  │ AI 📖 读取 chapters/   │  │ ← 工具调用气泡（灰底）
│  └────────────────────────┘  │
│                              │
│  ┌────────────────────────┐  │
│  │ AI 📖 读取 world_model │  │
│  └────────────────────────┘  │
│                              │
│  ┌─────────────────────────┐ │
│  │        你: 继续写第四章  │ │ ← 用户消息（紫底，右对齐）
│  └─────────────────────────┘ │
│                              │
│  ┌────────────────────────┐  │
│  │  AI: 好的，我先了解      │  │ ← AI 回复（白底，左对齐）
│  │  一下目前的剧情进展      │  │
│  └────────────────────────┘  │
│                              │
│  ┌────────────────────────┐  │
│  │ AI ✏️ 正在写入草稿…60% │  │ ← 工具调用（进行中）
│  └────────────────────────┘  │
├──────────────────────────────┤
│ [续写] [审核] [世界观] [文风] │ ← 动作按钮行
├──────────────────────────────┤
│ ┌────────────────────────┐   │
│ │ 输入指令，跟 AI 一起创作 │ ➤ │ ← 输入框
│ └────────────────────────┘   │
│ 模型: GPT-4o ▼               │ ← 模型选择器
└──────────────────────────────┘
```

**输入框交互**：

| 操作 | 行为 |
|------|------|
| Enter | 发送消息 |
| Shift+Enter | 换行 |
| 输入框为空时 | 发送按钮置灰 disabled |
| 发送后 | 清空输入框，滚动到最新消息 |
| AI 回复中 | 输入框 disabled + 显示「AI 正在回复…」+ 发送按钮变为 ■ 停止按钮 |

**动作按钮行**（位于消息列表和输入框之间）：

| 按钮 | 行为 |
|------|------|
| ✏️ 续写 | 预填系统 Prompt：「根据当前大纲和已有章节，续写下一章草稿」|
| 🔍 审核 | 预填系统 Prompt：「审查当前草稿，检查剧情冲突、人设一致性问题」|
| 🌍 世界观 | 预填系统 Prompt：「从已有章节中提取世界观设定，生成 world_model.md」|
| 🎨 文风 | 预填系统 Prompt：「分析已有章节的文风特征，生成 style_fingerprint.md」|

点击动作按钮后：
1. 自动切换到对应 Agent（后台逻辑切换，界面不显示 Tab）
2. 将预填 prompt 发送到聊天
3. 消息列表中显示该 prompt + AI 开始回复

**AI 回复流式展示**：

```
初始态:  AI 头像闪烁 + "正在思考…"
  ↓ 200ms
工具调用: 📖 读取 chapters/ 目录 · 已读 3 章
  ↓
工具结果: ✏️ 正在写入 chapter_draft.md … 已完成 60%
  ↓
文字输出: 逐 token 渲染，类似打字机效果
  ↓
完成: AI 头像停止闪烁，输出完整
```

**模型选择器**：

```
模型: [GPT-4o           ▼]
      ├── Claude Sonnet 4
      ├── Claude Haiku 3.5
      ├── DeepSeek V4
      └── GPT-4o
```

- 切换模型不会影响当前对话历史
- 切换后的下一条消息使用新模型
- 模型列表从设置中的 Provider 配置自动读取

       └──────────────────────────┘
```

## 10. 异常流全集

### 10.1 网络异常

| 场景 | 检测时机 | 用户反馈 | 恢复行为 |
|------|----------|----------|----------|
| API Key 无效 | 发送 AI 消息 / 测试连接 | Toast + 2s 后弹出设置 | 用户在设配置新 Key |
| LLM API 超时 | 请求超过 60s 无响应 | 红色消息气泡 + [重试] | 点击重试重新请求 |
| LLM API 返回错误 | 收到非 200 响应 | 红色消息气泡 + 错误原因 | 点击重试 |
| 网络断开 | 请求失败 (fetch error) | Toast「网络连接已断开」+ 所有 AI 按钮置灰 | 自动检测恢复后解除置灰 |

### 10.2 中途退出

| 场景 | 保存行为 | 恢复行为 |
|------|----------|----------|
| 切换页面 | 自动保存编辑器内容 | 回到该页时恢复 |
| 关闭窗口 | 自动保存编辑器 + 对话历史 | 下次启动回到上次所在页面 |
| 切换书籍 | 自动保存当前工作区 | 切回来时恢复 |
| 应用崩溃 | 文件系统已有实时写入（auto-save 30s）| 最多丢失 30s 内容 |

### 10.3 LLM 回复异常

| 异常 | 检测 | 界面表现 |
|------|------|----------|
| 模型返回格式错误 | Tool call 的 JSON 解析失败 | AI 消息红色边框 +「AI 返回格式异常，请重试」|
| 模型拒绝回答 | 内容策略拒绝 | 白色消息气泡 +「AI 拒绝了请求：{原因}」|
| Token 超限 | 请求前估算 + 响应截断检测 | 系统消息「📌 对话过长，已自动裁剪早期历史后重试」|
| 连续失败 3 次 | 失败计数器 | Toast「AI 服务暂时不可用，请过段时间再试」+ 按钮恢复为初始态 |

### 10.4 文件系统异常

| 异常 | 检测 | 行为 |
|------|------|------|
| 磁盘空间不足 | 写入文件时捕获 | Toast「保存失败：磁盘空间不足」|
| 文件被外部占用 | fs 写入失败 | Toast「文件被其他程序占用，请关闭后重试」|
| 权限不足 | 创建/删除文件时 | Toast「操作失败：权限不足，请以管理员身份运行」|
| Git 仓库损坏 | git 命令报错 | Toast「Git 仓库异常，尝试修复中…」+ 后台尝试 `git fsck` |

### 10.5 操作冲突

| 场景 | 用户反馈 |
|------|----------|
| 连续点击续写 2 次 | 第二个被加入队列，前一个完成后自动执行 |
| 归档进行中退出页面 | 强制等待归档完成（最多 3s），完成后跳转 |
| 编辑器内容和文件系统不一致（外部修改）| Toast「文件已在外部被修改，是否重新加载？」[重新加载] [保留当前] |

## 11. 动效说明

### 11.1 页面切换

| 方向 | 动效 | 时长 | 缓动 |
|------|------|------|------|
| 侧边栏点击切换 | 内容区域交叉淡入淡出（opacity） | 200ms | ease |
| 作品库→创作台 | 从右向左 slide-in | 300ms | ease-out |
| 创作台→版本管理 | 从右向左 slide-in | 300ms | ease-out |
| 弹窗打开 | 背景遮罩渐入 + 内容缩放从 0.95→1 | 200ms | ease-out |
| 弹窗关闭 | 背景遮罩渐出 + 内容缩放从 1→0.95 | 150ms | ease-in |

### 11.2 组件动效

| 组件 | 动效 | 时长 |
|------|------|------|
| 按钮悬停 | 背景色过渡 | 150ms |
| 按钮点击 | scale(1)→scale(0.98)→scale(1) | 100ms |
| Toast 出现 | translateY(-20)→0 + opacity 0→1 | 250ms |
| Toast 消失 | translateY(0)→-20 + opacity 1→0 | 250ms |
| 卡片悬停 | translateY(0)→(-2) + box-shadow 加深 | 200ms |
| 骨架屏 | 从左到右扫光 shimmer | 1.5s 循环 |
| 文件树展开 | max-height 从 0 → 实际值 | 200ms |
| AI 流式文字 | 逐 token 追加，无过渡 | 实时 |
| 工具调用气泡 | 从上方淡入 | 150ms |
| 状态标记切换 | 图标颜色过渡 | 200ms |
| 下拉菜单展开 | opacity 0→1 + translateY(-4)→0 | 150ms |

### 11.3 加载状态

| 场景 | 动效 | 说明 |
|------|------|------|
| AI 回复中 | AI 头像脉冲闪烁（opacity 0.6→1 循环） | 持续至回复完成 |
| 文件加载 | 编辑器显示骨架屏 | 加载完成后内容淡入 |
| Git 操作 | 按钮显示旋转图标 | 操作完成后切换状态 |
| 保存文件 | 顶部栏显示「已保存」文案淡入→2s→淡出 | 自动保存不显示 |
| 导入进度 | 卡片底部进度条动画 | 从 0 → 100% |

### 11.4 过渡与反馈

| 反馈 | 动效 |
|------|------|
| 操作成功 | 绿色 Toast +（可选）完成图标 |
| 操作失败 | 红色 Toast +（可选）按钮恢复初始态 |
| 表单校验失败 | 输入框红色边框抖动（shake 50ms × 3）|
| 拖拽分隔条激活 | 分隔条从透明变为紫色 (#7c5cfc) |
