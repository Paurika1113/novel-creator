# Novel Creator — 四周开发与交付计划

> 基于 PRD V 2.0.0 · 2026-06-01 → 2026-06-28

---

## 技术选型

| 层 | 选型 | 理由 |
|---|------|------|
| 桌面框架 | **Electron 34+** | 本地文件系统访问，成熟生态 |
| 前端框架 | **React 19 + TypeScript 5** | 类型安全，生态成熟 |
| 构建 | **Vite** | HMR 快，Electron + Vite 组合成熟 |
| 样式 | **Tailwind CSS v4** | 原子化类名，暗色/亮色主题原生支持 |
| 状态管理 | **Zustand** | 轻量，无 boilerplate，适合多 store 拆分 |
| Markdown 编辑器 | **TipTap** (ProseMirror 内核) | WYSIWYG + Markdown 源码切换 |
| 本地 Git | **isomorphic-git** | 纯 JS，无需 Git CLI，主进程运行 |
| LLM 集成 | **OpenAI 兼容 SDK** (function calling) | 统一接口支持 Claude / DeepSeek / GPT |
| 文件监听 | **chokidar** | 本地文件变更检测 |
| 打包分发 | **electron-builder + NSIS** | Windows 安装包 |
| 原型工具 | **HTML 原型（延续现有方式）** | 与代码紧密衔接，无工具切换成本 |

**核心类型示例（Persona 相关）**：

```typescript
interface Persona {
  id: string
  name: string
  createdAt: string
  sourceBookIds: string[]
  analysisStatus: 'idle' | 'analyzing' | 'completed' | 'failed'
  styleProfile: StyleProfile
  manualOverrides: Record<keyof StyleProfile, boolean>
}

interface StyleProfile {
  lexical: string          // 语言层描述
  narrative: string        // 叙事层描述
  structural: string       // 结构层描述
  stylistic: StylisticTags // 风格标签
}

interface WaterLevel {
  mild: number    // 默认 0.4
  moderate: number // 默认 0.7
  deep: number    // 默认 0.85
}
```

## 风险与缓解

| 风险 | 概率 | 影响 | 缓解 |
|------|------|------|------|
| LLM function calling 行为不稳定（模型不按 schema 调用工具）| 中 | 高 | 主要使用 Claude（tool use 最稳定）；DeepSeek 作为备选；加验证层，不符合 schema 时重试 |
| 事件图谱分析质量不稳定（LLM 产出 events.json 不够准确）| 中 | 中 | 提供清晰的 prompt 模板 + 输出 schema constraints；用户可手动编辑 active_elements.md |
| 记忆压缩后内容失真 | 低 | 中 | T1/T2/T3 压缩文件也可编辑，用户发现失真时可手动修正 |
| 百章级大书的 performance | 低 | 中 | isomorphic-git 纯文本仓库压力较低；事件图谱和压缩管线设计为非阻塞后台 |
| 分析管线 token 消耗大 | 中 | 低 | 分析管线在后台非阻塞执行，用户无感知；提供分析精度选项让用户权衡速度和质量 |
| 4 周时间紧 | 高 | 中 | 见下方降级方案 |

## 应急降级方案

如果某周滞后，以下功能可安全后移：

1. **文风 Agent** → 降级为仅使用续写 Agent 内置的文风约束（区块 A 的声纹描述），不作为独立 Tab
2. **事件图谱 + 故事线程** → 降级为纯距离压缩（不使用线程加权），等 v1.1.0 再完善
3. **审核 Agent 深度审核** → 降级为仅检查世界观一致性 + 前文连续性（保留前三级审核，跳过文风和文本质量）
4. **知识文件变更通知** → 降级为用户手动触发重读（在聊天面板加一个"刷新上下文"按钮）
5. **设置页压缩敏感度滑条** → 降级为固定 40% 阈值
6. **用户手册** → 发布后补
