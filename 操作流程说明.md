# Novel Creator 操作流程说明

> 本文档基于实际项目操作经验整理，涵盖从项目启动到知识文件生成的完整流程。

---

## 一、项目启动

### 1.1 启动开发服务器

```bash
# 进入项目目录
cd e:\AIzy\OH-WorkSpace\novel-creator

# 安装依赖（首次使用）
npm install

# 启动开发服务器
npm run dev
```

服务器默认运行在 `http://localhost:5174/`，浏览器会自动打开。

### 1.2 界面布局说明

应用采用三栏式布局：
- **左侧导航栏**：作品库、创作台、版本管理、作者身份、设置
- **中间文件树**：章节、草稿、知识文件列表
- **右侧编辑器**：文本编辑和 AI 对话区域

---

## 二、API 配置（首次使用必须）

### 2.1 进入设置

1. 点击左下角 **⚙️ 设置** 按钮
2. 选择 **供应商** 标签页

### 2.2 配置 OpenAI 兼容接口

| 配置项 | 说明 | 示例 |
|:---|:---|:---|
| 名称 | 自定义供应商名称 | 自定义 |
| Base URL | API 基础地址 | `https://opencode.ai/zen/v1` |
| API Key | 你的 API 密钥 | sk-... |
| 模型 | 使用的模型名称 | `deepseek-v4-flash-free` |

### 2.3 保存配置

点击 **保存** 按钮，配置会自动持久化到本地存储。

> ⚠️ **注意**：配置完成后刷新页面，确认下拉框显示的是你配置的模型（如 `deepseek-v4-flash-free`），而非默认的 `gpt-4o`。

---

## 三、创建书籍

### 3.1 进入作品库

1. 点击左侧导航栏 **📁 作品库**
2. 点击右上角 **＋ 新建** 按钮

### 3.2 填写书籍信息

| 字段 | 填写内容 | 示例 |
|:---|:---|:---|
| 书名 | 小说完整名称 | 《穿成恶毒女配后我驯化了疯批暴君》 |
| 类型 | 题材分类 | 穿书古言 |
| 简介 | 一句话描述 | 穿书女配驯化疯批暴君的故事 |
| 主角 | 主角姓名 | 苏辞意 |

### 3.3 创建并触发知识初始化

点击 **创建** 按钮后，系统会自动：
1. 创建书籍
2. 弹出 **🤖 知识文件初始化** 对话框

---

## 四、AI 知识文件初始化（核心流程）

### 4.1 对话流程概述

AI 会按以下话题顺序逐一提问，你需要根据小说设定回答：

| 顺序 | 话题 | 讨论重点 |
|:---|:---|:---|
| 1 | 世界类型与核心设定 | 时代背景、世界观规则、核心机制（如穿书、剧情修复力） |
| 2 | 力量体系 | 武力等级、特殊能力、毒术/蛊术体系 |
| 3 | 地理与势力分布 | 地图、各方势力、政治格局 |
| 4 | 主角与关键人物 | 主角背景、性格、开局处境 |
| 5 | 人物关系与互动 | 主角与其他角色的关系演变 |
| 6 | 故事主线和核心冲突 | 终极目标、核心矛盾、循环机制 |
| 7 | 篇章规划 | 分卷结构、章节数、高潮节点 |

### 4.2 回答技巧

- **详细具体**：不要只给概括，要给出具体的设定细节
- **主动延伸**：如果 AI 没有问到但重要的设定，可以主动补充
- **纠正错误**：如果 AI 理解有误，及时指出并更正
- **控制节奏**：某个话题讨论充分后，可以说"进入下一个话题"

### 4.3 完成讨论

当所有话题讨论完毕后，点击底部 **🎯 完成讨论并生成知识文件** 按钮。

### 4.4 处理生成错误

如果显示 `BodyStreamBuffer was aborted` 错误：
1. 不要关闭对话框
2. 直接再次点击 **完成讨论并生成知识文件** 按钮
3. 等待生成完成（可能需要 30-60 秒）

### 4.5 生成完成

生成成功后，会显示：
- ✅ 知识文件生成完成
- 文件预览（世界观设定、全书总纲、灵感笔记）
- **进入创作台** 按钮

---

## 五、进入编辑器

### 5.1 查看知识文件

点击 **进入创作台** 后，在左侧文件树的 **📚 知识** 分类下可以看到：

| 文件 | 用途 |
|:---|:---|
| 🌍 世界观设定 | 故事背景、规则、地理、势力 |
| 📋 总纲 | 全书主线剧情大纲 |
| 💡 灵感笔记 | 碎片化创意、临时想法 |
| 🎨 文风画像 | AI 模仿的目标风格 |
| 📜 卷纲 | 分卷剧情结构 |
| 📑 章纲 | 细粒度章节大纲 |
| 📊 状态卡 | 角色状态追踪 |
| ⚠️ 错误档案 | AI 生成错误记录 |
| 📝 摘要 | 前文内容压缩 |

### 5.2 查看和编辑文件

1. 点击文件树中的任意文件
2. 在右侧编辑器中查看内容
3. 直接编辑修改
4. 按 `Ctrl+S` 保存

---

## 六、开始创作

### 6.1 创建章节

1. 点击文件树中 **📁 章节** 文件夹
2. 点击 **➕ 写下一章** 按钮
3. 输入章节标题

### 6.2 AI 续写

1. 在底部输入框输入指令（如"续写下一章"）
2. 按 `Enter` 发送
3. AI 会根据知识文件和前文内容生成章节内容

### 6.3 常用快捷操作

| 快捷键 | 功能 |
|:---|:---|
| `Ctrl+S` | 保存当前文件 |
| `Enter` | 发送 AI 指令 |
| `Shift+Enter` | 换行 |

## 七、常见问题

### Q1: 模型显示为 gpt-4o 怎么办？

**原因**：settingsStore 没有正确持久化，刷新后恢复默认值。

**解决**：
1. 进入 **设置 → 供应商**
2. 确认你的配置是否正确保存
3. 重新保存一次
4. 刷新页面验证

### Q2: 知识文件生成失败怎么办？

**原因**：网络超时或流式传输中断。

**解决**：
1. 保持对话框打开
2. 再次点击 **完成讨论并生成知识文件**
3. 等待更长时间（最长可能需要 1-2 分钟）

### Q3: 如何修改已生成的知识文件？

**解决**：
1. 在创作台左侧文件树中找到对应文件
2. 点击打开
3. 直接编辑内容
4. `Ctrl+S` 保存

### Q4: 书籍类型填写错误怎么办？

**解决**：
1. 当前版本不支持直接修改书籍信息
2. 可以删除书籍后重新创建
3. 或在知识文件的"世界观设定"中补充正确的类型说明

## 八、最佳实践

1. **先完善知识再创作**：知识文件越详细，AI 生成的内容越准确
2. **定期保存**：虽然应用有自动保存，但重要修改后建议手动 `Ctrl+S`
3. **利用灵感笔记**：随时记录新想法，避免遗忘
4. **更新错误档案**：AI 生成出错时记录下来，帮助后续优化
5. **分阶段细化**：先完成粗纲，再逐步细化到章纲
